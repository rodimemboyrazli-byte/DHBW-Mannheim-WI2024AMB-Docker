## Setup:
1. Docker installieren: [Docker Desktop](https://www.docker.com/products/docker-desktop/)
oder CLI-Version: [Docker Engine](https://docs.docker.com/engine/install/)
2. Dieses Repository forken
3. Repository klonen: `git clone <dein_fork_url>`
4. In das Verzeichnis wechseln: `cd <Ordnername>`
5. Ordner `apis` anschauen, um die APIs zu verstehen

## Aufgabe: Docker-Container für APIs erstellen

1. Neuen Ordner erstellen: `solution_<Name>`
2. Den Inhalt des Ordners `apis` in deinen neuen Ordner kopieren
3. Für jede API die Dockerfile erstellen
4. In deinem Ordner eine `docker-compose.yml` erstellen, um alle APIs zu starten
5. Testen, ob alle APIs korrekt starten und ob die Endpunkte erreichbar sind
6. Dokumentation erstellen (README.md)

## Abgabe:
1. Forked Repository auf GitHub pushen
2. Pull Request erstellen, um deine Lösung zu präsentieren
3. In der Pull Request eine kurze Beschreibung deiner Lösung und eventuelle Herausforderungen hinzufügen
